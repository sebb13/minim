Media

Le module Media permet de gérer les médias utilisés sur votre application.

Il ne sert à rien seul, c'est un module de dépendence.

La médiathèque de votre site/application !

Vous pouvez gérer des :

  • images
  • fichiers audio
  • fichiers vidéo
  • flux
  • iframe
  • PDF
  • ZIP

Configuration

Vous pouvez configurer :

  • La taille maximum de vos images.
  • La qualité (compression) de vos miniatures.
  • Le nombre de média à afficher par page.
  • Ce module nécessite plusieurs tables en base de données

Installation

Créez et peuplez les tables nécessaires au fonctionnement du module (fichier .sql dans /data).

Comme pour tous les modules, n'oubliez pas de :

  • Copier les classes PHP (présentes dans /lib) dans le répertoire Core/lib/Modules/ de votre site/application.
  • Ajouter les fichiers css et js dans votre configuration (/Core/data/concat_js.xml et /Core/data/concat_css.xml) afin qu'ils soient pris en compte.
  • Régénérer le cache afin que le module soit pris en compte.

L'accueil de ce plugin vous permet de :

  • Gérer les médias actifs.
  • Gérer les médias archivés.
  • Ajouter un nouveau média.
  • Rechercher un média.
  • Visualiser les médias actifs et y accéder (par type).
  • Gérer la configuration du module.

La base de données se paramètre dans la configuration globale.

Dashboard

Le service du module fourni une méthode getDashboard() qui permet d'afficher sur l'accueil du backoffice un rapide état des lieux.

Ajout d'un média

Chaque média a ses spécificités, ici l'exemple des images :

  • url
  • src
  • alt_FR (la langue de référence dans cet exemple)
  • media_description_FR (la langue de référence dans cet exemple)
  • title_FR (la langue de référence dans cet exemple)
  • Traductions (Les balises alt, title et meta-description dans les autres langues disponibles sur l'application)
  • Si le média est téléchargeable depuis le site internet

Edition d'un média

Ici pour une image, vous pouvez modifier les informations saisies lors de l'ajout :

  • url
  • src
  • alt_FR (la langue de référence dans cet exemple)
  • media_description_FR (la langue de référence dans cet exemple)
  • title_FR (la langue de référence dans cet exemple)
  • Traductions (Les balises alt, title et meta-description dans les autres langues disponibles sur l'application)
  • Si le média est téléchargeable depuis le site internet

Vous pouvez également créer dynamiquement une page dédiée à ce média (pour le partager simplement par exemple).

Gestionnaire

Ici le gestionnaire de médias pour l'audio.

Cette interface vous permet :

  • De changer de type de média à gérer.
  • D'accéder aux opérations possibles par type de média (génération de miniatures, etc).
  • D'accéder au formulaire d'ajout en fonction du type de média.
  • D'accéder aux médias archivés.
  • D'accéder au formulaire d'édition du média de votre choix.
  • D'archiver des médias.

Traitements par lot

Pour les images, vous pouvez :

  • Consulter les tailles de miniatures disponibles.
  • Générer une nouvelle taile de miniature
  • Régénérer toutes les miniatures dans toutes les tailles.
  • Rechercher un autre média.

Recherche

Le module fourni un moteur de recherche en bas de chaque bloc d'opération sur les médias

Vous indiquez le type de média recherché (par défaut il s'agit du type en cours) et les mots clés de votre recherche.
